# AWS Backup CloudFormation Lab

## English

## Overview

This project demonstrates how to deploy a generic AWS Backup strategy using CloudFormation.

The solution includes:

- AWS Backup Vault
- AWS Backup Plan
- Daily backup rule
- Retention policy
- Cold storage lifecycle
- IAM role for AWS Backup
- Tag-based resource selection
- Operational backup validation documentation

This repository is a generic and anonymized technical lab. It does not contain proprietary information, production data, real AWS account IDs, real ARNs, credentials, internal company names or confidential documentation.

## Objective

The objective of this lab is to show how backup automation can be implemented as Infrastructure as Code using AWS CloudFormation.

The project also includes operational documentation related to failed backup states, RPO/RTO definitions and backup validation controls.

## Repository Structure

aws-backup-cloudformation-lab/
├── README.md
├── templates/
│   └── aws-backup-plan.yaml
├── parameters/
│   └── parameters-demo.json
├── docs/
│   ├── deployment-guide.md
│   ├── backup-job-status.md
│   └── rpo-rto-matrix.md
└── checklists/
    └── backup-validation-checklist.md

## Main Components

| Component | Description |
|---|---|
| AWS Backup Vault | Stores recovery points generated by backup jobs |
| AWS Backup Plan | Defines backup rules and lifecycle configuration |
| Backup Rule | Schedules backup execution |
| IAM Role | Allows AWS Backup to perform backup and restore operations |
| Tag-based Selection | Selects resources using the tag `Backup = Enabled` |
| Retention Policy | Defines how long recovery points are kept |
| Cold Storage Lifecycle | Moves recovery points to cold storage after a defined period |

## Default Backup Configuration

| Setting | Value |
|---|---:|
| Backup schedule | Daily at 02:00 UTC |
| Tag selection | `Backup = Enabled` |
| Move to cold storage after | 7 days |
| Delete recovery point after | 35 days |
| Stack name | `demo-aws-backup-stack` |

## Deployment

Validate the CloudFormation template:

aws cloudformation validate-template \
  --template-body file://templates/aws-backup-plan.yaml

Deploy the stack:

aws cloudformation deploy \
  --template-file templates/aws-backup-plan.yaml \
  --stack-name demo-aws-backup-stack \
  --parameter-overrides file://parameters/parameters-demo.json \
  --capabilities CAPABILITY_NAMED_IAM

## Operational Documentation

This repository also includes:

- Backup job status and failed state handling
- RPO and RTO matrix
- Backup validation checklist
- Deployment guide

## Security Disclaimer

This project uses only demo values.

Do not upload:

- Real AWS account IDs
- Real ARNs
- Access keys
- Secrets
- Production tags
- Internal company names
- Customer information
- Production documentation
